# 無界茶台 CRM 后台设计方案

## 一、需求分析

### 1.1 业务流程
```
客户提交预约 → 系统分配销售 → 销售跟进 → 成交/关闭
```

### 1.2 角色权限
| 角色 | 权限 |
|------|------|
| 管理员 | 查看所有数据、分配客户、管理销售、查看统计 |
| 销售 | 查看自己的客户、跟进客户、更新状态 |

## 二、数据库设计

### 2.1 新增表

#### sales (销售表)
```sql
CREATE TABLE sales (
    id INT PRIMARY KEY AUTO_INCREMENT,
    user_id INT NOT NULL,              -- 关联 users 表
    name VARCHAR(50) NOT NULL,         -- 销售姓名
    phone VARCHAR(20),                 -- 销售电话
    email VARCHAR(100),
    wechat VARCHAR(50),                -- 微信号
    daily_limit INT DEFAULT 10,        -- 每日分配上限
    is_active BOOLEAN DEFAULT TRUE,    -- 是否在职
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id)
);
```

#### booking_assignments (客户分配表)
```sql
CREATE TABLE booking_assignments (
    id INT PRIMARY KEY AUTO_INCREMENT,
    booking_id INT NOT NULL,
    sales_id INT NOT NULL,
    assigned_by INT,                   -- 分配人（管理员）
    assigned_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    status ENUM('active', 'transferred', 'closed') DEFAULT 'active',
    notes TEXT,
    FOREIGN KEY (booking_id) REFERENCES bookings(id),
    FOREIGN KEY (sales_id) REFERENCES sales(id),
    FOREIGN KEY (assigned_by) REFERENCES users(id)
);
```

#### follow_ups (跟进记录表)
```sql
CREATE TABLE follow_ups (
    id INT PRIMARY KEY AUTO_INCREMENT,
    booking_id INT NOT NULL,
    sales_id INT NOT NULL,
    contact_type ENUM('phone', 'wechat', 'visit', 'other'), -- 联系类型
    contact_result TEXT,               -- 沟通结果
    next_follow_up DATE,               -- 下次跟进时间
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (booking_id) REFERENCES bookings(id),
    FOREIGN KEY (sales_id) REFERENCES sales(id)
);
```

## 三、API 接口设计

### 3.1 销售管理（管理员）

| 接口 | 方法 | 说明 |
|------|------|------|
| /api/admin/sales | GET | 获取销售列表 |
| /api/admin/sales | POST | 添加销售 |
| /api/admin/sales/:id | PUT | 更新销售信息 |
| /api/admin/sales/:id/assignments | GET | 获取销售客户分配情况 |

### 3.2 客户分配（管理员）

| 接口 | 方法 | 说明 |
|------|------|------|
| /api/admin/bookings/unassigned | GET | 获取未分配客户 |
| /api/admin/bookings/assign | POST | 分配客户给销售 |
| /api/admin/bookings/reassign | POST | 重新分配客户 |

### 3.3 销售工作台

| 接口 | 方法 | 说明 |
|------|------|------|
| /api/sales/dashboard | GET | 销售工作台数据 |
| /api/sales/bookings | GET | 获取我的客户 |
| /api/sales/bookings/:id | GET | 客户详情 |
| /api/sales/bookings/:id/followup | POST | 添加跟进记录 |
| /api/sales/bookings/:id/status | PUT | 更新客户状态 |

## 五、自动分配策略

### 5.1 分配规则
1. **轮询分配**：按顺序分配给每个销售
2. **负荷均衡**：优先分配给当前客户少的销售
3. **能力匹配**：根据销售专长分配（如高端客户分配给资深销售）

### 5.2 每日上限
- 每个销售有每日分配上限（默认10个）
- 超过上限后不再分配，进入待分配池

## 六、通知机制

### 6.1 销售通知
- 新分配客户时：微信/短信通知
- 客户提交新预约时：管理员通知

### 6.2 跟进提醒
- 超过24小时未跟进：提醒销售
- 设置的下次跟进时间到期：提醒销售

## 七、数据统计

### 7.1 销售统计
- 客户转化率
- 平均跟进次数
- 平均成交周期

### 7.2 整体统计
- 每日新增客户
- 每日成交数
- 各产品系列转化率
